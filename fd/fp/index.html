                  # 函数式编程

                  # 纯函数

                  纯函数是这样一种函数，即相同的输入，永远会得到相同的输出，而且没有任何可观察的副作用

                  好处：

                  • 可缓存性（Cacheable），例如 memoize 函数
                  • 可移植
                  • 可测试
                  • 并行代码

                  可以通过延迟执行的方式把不纯的函数转换为纯函数

                  # 柯里化

                  柯里化：只传递给函数一部分参数来调用它，让它返回一个函数去处理剩下的参数

                  简单实现

                  // 柯里化的原理只是把函数的参数缓存起来，待到时机（满足原函数的参数个数），调用函数
// 传入第二个参数 len， 明确函数的参数个数，防止es6中的默认值和剩余参数的影响
function curry(fn, len) {
	if (typeof fn !== 'function') throw new Error("curry's fist parameter must be function")
	function _c(restArg, args) {
		return restArg <= 0
			? fn.apply(null, args)
			: function() {
				var arg = [].slice.call(arguments)
				return _c(restArg - arg.length, args.concat(arg))
			}
	}
	return _c(len || fn.length, [])
}

// es6 箭头函数省去this的变更
const curry = (fn, n = fn.length, args = []) => n <= 0 ? fn(...args) : (...args1) => curry(fn, n - args1.length, [...args, ...args1]);
 
//举例
const add = curry((a, b, c, d, e) => a + b + c + d + e);
 
console.log(add(1, 2)(3, 4)(5));

                  # 函数组合

                  组合多个函数为一个新的函数，同时满足组合的可结合

                  结合律：compose(f, compose(g, h)) == compose(compose(f, g), h) == compose(f, g, h)

                  简单实现：

                  // 先执行最右边的函数
function compose (fns) {
	var args = Array.isArray(fns) ? fns : [].slice.call(arguments)
	return function () {
		return args.reduceRight((a, b) => {
			return b(a)
		}, [].slice.call(arguments))
	}
}

// 先执行左边的函数
function pipe (fns) {
	var args = Array.isArray(fns) ? fns : [].slice.call(arguments)
	return function () {
		return args.reduce((a, b) => {
			return b(a)
		}, [].slice.call(arguments))
	}
}

                  Pointfree

                  Pointfree风格：只合成运算过程，不依赖所处理的值，在函数式编程中，最明显的就是无参数

                  # 声明式代码

                  函数式编程是声明式编程，而非命令式编程，它指明的是做什么，不是怎么做

                  例子：

                  // 命令式
var authenticate = function(form) {
  var user = toUser(form);
  return logIn(user);
};

// 声明式
var authenticate = compose(logIn, toUser);

                  # 容器

                  一个容器就是范畴学里面的一个范畴，可以包含任何值，同时提供相应的函数(态射)

                  class Container () {
	// 构造器，我们使用 __value 保存值
	constructor (v) {
		this.__value = v
	}

	// 使用 of 构造容器，省去 new 关键字
	static of (v) {
		return new Container（v）
	}

	// 定义 map 是函数可以操作值，返回一个新的容器以保持纯洁，和链式调用
	// 让容器自己去运用函数能使函数的调用变得抽象
	map (f) {
		return Container.of(f(this.__value))
	}
}

                  # 辅助函数

                  # map函数

                  柯里化容器的map函数，方便后续的 compose 组合，符合 pointTree 风格

                  // map :: Functor f => (a -> b) -> f a -> f b
var map = curry(function(f, Functor) {
  return Functor.map(f)
})

                  # maybe函数

                  对容器里的值来说，在组合过程中，如果我们想返回一个自定义的值然后还能继续执行后面的代码的话，我们需要借助 maybe 函数 maybe函数避免在函数式编程中使用 if/else 语句，当然也可以使用 Either 函子

                  //  maybe :: b -> (a -> b) -> Maybe a -> b
var maybe = curry(function(x, f, m) {
  return m.isNull() ? x : f(m.__value)
})

                  # chain函数

                  chain函数是 Monad 函子中的flatMap的提取

                  //  chain :: Monad m => (a -> m b) -> m a -> m b
var chain = curry(function(f, m) {
  return m.map(f).join()  // 或者 compose(join, map(f))(m)
})

                  # ap函数

                  ap函数是 applicative 函子中的ap的提取

                  //  ap :: Applicative F => (a -> m b) -> m a -> m b
var ap = curry(function(v, F) {
  return F.map(v.__value)
})

                  特性：F.of(x).map(f) == F.of(f).ap(F.of(x)) 即：map 一个 f 等价于 ap 一个值为 f 的 functor

                  # 函子

                  函子 (functor) 是实现了 map 函数并遵守一些特定规则的容器类型，即函子也是一个容器

                  一些常用的函子

                  # maybe

                  Maybe 函子通常用来做空值检测，或者错误处理

                  class Maybe extends Container {
	static of (v) {
		return new Maybe(v)
	}

	isFalsify () {
		return !!this.__value
	}

	isNull () {
		return this.__value == null
	}

	map (f) {
		return this.isNull() ? Maybe.of(null) : Maybe.of(f(this.__value))
	}
}

                  # Either

                  Either 函子内部有两个值：左值（Left）和右值（Right）。右值是正常情况下使用的值，左值是右值不存在时使用的默认值。

                  class Either extends Container {
	constructor (left, right) {
		this.left = left
		this.right = right
	}

	static of (l, r) {
		return new Either(l, r)
	}

	map (f) {
		return this.right 
			? Either.of(this.left, f(this.right))
			: Either.of(f(this.left), this.right)
	}
}

                  # Monad

                  Monad 函子的作用是，总是返回一个单层的函子，monad 的执行会等待上一个 monad 执行结束，类似于promise的then方法

                  class Monad extends Container {
	join () {
		return this.__value
	}
	flatMap (f) {
		return this.map(f).join()
	}
}

                  # applicative

                  ap 就是把一个 functor 的函数值应用到另一个 functor 的值上的一个函数

                  class Ap extends Container {
	ap (F) {
		// return Ap.of(this.__value(F.__value))
		return Ap.of(F.map(this.__value))
	}
}
