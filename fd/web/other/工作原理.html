          # 浏览器的工作原理

          # 认识浏览器

          浏览器的功能是将用户搜索的界面资源显示出来

          简单来说浏览器可以分为两部分，界面和内核

          界面包含菜单，导航，工具栏等等，提供给用户操作和设置

          内核才是浏览器的核心，常见的浏览器内核可以分这四种：Trident、Gecko、 Presto、Webkit

          # 浏览器的主要进程

          js是单线程的，但是浏览器是多线程的，每打开一个tab，就会创建一个独立的浏览器进程(渲染进程)

          浏览器多进程的优点：

          • 不同tab相互独立，避免一个出现问题影响整个浏览器
          • 避免第三方插件出现问题影响整个浏览器
          • 充分利用系统的多核优势，提高效率

          多进程就意味着需要给每一个进程分配内存和资源

          为什么js是单线程呢? js是一个脚本语言，主要用于实现用户与浏览器的交互行为。设计为单线程可以简化问题

          浏览器主要包含的进程：

          1. Browser进程：浏览器的主进程（负责协调、主控），只有一个。作用有
          • 负责浏览器界面显示，与用户交互。如前进，后退等
          • 将Renderer进程得到的内存中的Bitmap，绘制到用户界面上
          • 负责各个页面的管理，创建和销毁其他进程
          • 网络资源的管理，请求下载等

          1. GPU进程：用于3D绘制等

          2. 浏览器渲染(Renderer)进程：默认每个Tab页面一个进程，互不影响。主要作用有

          • 页面渲染
          • 脚本执行
          • 事件处理
          1. 第三方插件进程：每种类型的插件对应一个进程，仅当使用该插件时才创建

          # 浏览器渲染进程

          渲染进程是浏览器的核心，是如何将拿到的资源渲染为我们看到的页面的核心，渲染进程的主要线程有：

          1. GUI渲染线程: 页面的渲染和重绘

          2. JS引擎: 解析js脚本，运行代码

          3. 事件处理线程: eventLoop

          4. 定时触发器线程

          5. 异步http请求线程

          # web页面的展示

          我们打开浏览器，在地址栏输入地址，到页面显示的这一过程中发生了什么事情呢

          1. 资源的请求 浏览器主进程发送一个请求 域名解析: 详细的域名解析 tcp链接

          2. 页面解析

            1. 浏览器主进程获取页面后，通过RendererHost接口传递给浏览器渲染进程
            2. 浏览器渲染进程传递给GUI渲染线程
            3. GUI渲染线程解析html
              1. 解析html建立DOM树
              2. 解析css构建CSSOM树
              3. 将DOM树和CSSOM树合成render树
              4. 布局render树（Layout/reflow），负责各元素尺寸、位置的计算
              5. 绘制render树（paint），绘制页面像素信息
              6. 复合图层，浏览器会将各层的信息发送给GPU，GPU会将各层合成（composite），显示在屏幕上。
            4. js解析
              1. JS引擎解析js代码，词法解析和语法解析
              2. 运行代码
              3. 遇到http请求，异步http请求线程去请求资源
              4. 遇到异步函数，事件处理线程将回调函数推入事件队列
              5. 遇到计时器，定时触发器线程将回调函数推入事件队列中，同时精准计时
            5. EventLoop

          3. 回流、重绘
