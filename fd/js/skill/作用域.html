          • 变量对象(VO)
          • 作用域链(scope)

          # 变量对象

          变量对象是执行上下文中的一个特殊的属性，它存储着在上下文中声明的以下内容：

          • 变量声明
          • 函数声明
          • 函数的形参

          变量对象(VO)是在代码未执行之前，代码编译后形成的对象，这个对象无法访问，但是进入执行阶段之后，变量对象转变为了活动对象(AO)

          全局上下文中的变量对象就是自己，同时包含了全局属性，window，Math等，这就是我们可以直接调用他们的原因

          在函数执行上下文中，变量对象是不能直接访问的，此时由活动对象扮演变量对象的角色 活动对象(AO)除了包含变量对象(VO)，还包含函数的形参(arguments对象)和实参

          参考链接：http://www.cnblogs.com/TomXu/archive/2012/01/16/2309728.html

          # 变量的重复宣告

          var a = 1
var a
console.log(a) // 1

          上述情况就是变量的重复宣告，这和变量的重新定义是有区别的 JS引擎对于重复宣告的规定是以最近的变量赋值作为变量执行时的值

          以函数内形参和变量同名的情况来分析

          var a = { n: 1 }

(function (a) {
    console.log(a.n)    // 函数实参指向a的引用，a.n -> 1
    a.n = 3             // 对引用a.n赋值，此时 a.n -> 3
    var a = { n: 2 }    // 这里a变量的定义会提升，和函数的形参构成重复宣告，重复宣告使用a的引用的值；执行到这里，对变量a重新赋值，此时 a.n -> 2
    console.log(a.n)
})(a)

console.log(a.n)

// 打印顺序 1 -> 2 -> 3

          # 变量的作用域

          变量的作用域就两种：全局变量和局部变量

          全局作用域：最外层函数定义的变量拥有全局作用域，即对任何内部函数来说，都是可以访问的

          局部作用域：局部作用域一般只在固定的代码片段内可访问到，而对于函数外部是无法访问的

          变量定义的时候会有变量提升的现象，就是所有的变量定义都会被提升到执行上下文的顶部定义 函数的变量提升在普通变量的定义提升之前

          函数内部声明变量的时候，一定要使用var命令。如果不用的话，你实际上是在全局对象时添加了一个属性(并不是声明一个变量，可以用delete实验一下)

          变量的作用域会沿着局部作用域，局部作用域链，全局作用域和全局作用域链的顺序去访问
