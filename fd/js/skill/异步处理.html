        # 异步处理

        # 常见的异步场景

        • dom事件监听
        • 网络请求
        • 计时器
        • 数据监听
        • 任务等待

        由于JS是单线程执行的，在处理以上场景的问题上，如果没有引入异步，页面性能将会难以解决

        异步直观理解是中断程序A去执行B之后再回到断点处继续执行A，本质上这就是一种执行权的借还

        # 常见的异步处理方式

        1. 事件回调 事件回调在JS中随处可见，比如：定时器；回调是最常用的解决异步的方法

        使用事件回调处理异步，简单易解，但是回调过多，会出现回调地狱，而且不方便阅读

        1. 发布/订阅模式(事件监听) 发布/订阅模式采用事件驱动模式，和事件回调机制一样，发布/订阅模式需要提供回调函数（订阅），不同的是事件回调机制是自行触发，而发布/订阅模式把触发权限交给了开发者，因此你可以选择在任意时刻触发回调（发布）。 web开发中的dom事件就属于此类，该模式常暴露的方法有：
        • on或者addEventListener: 订阅
        • observe: 发布
        • removeListener: 取消订阅
        1. Promise Promise的本质依然是事件回调，只是事件回调的一个语法糖； Promise在时间回调的基础上，在内部保存了回调的状态，回调函数和执行结果，使用then方法添加回调，同时把异步函数的执行结果作为回调的值；使用catch方法添加函数调用异常的回调，同时把异常作为异常回到的参数； 然后通过开发者触发resolve或者reject来控制回调的成功与否

        简单的说，它的思想是：每一个异步任务返回一个promise对象，该对象有一个then方法，允许指定回调函数，有一个catch方法用来捕捉异常

        使用promise模式可以链式添加回调，代码更清晰

        1. Generator (es6) Generator 函数其实是一个状态机，封装了多个内部状态。 Generator 函数提供了一种机制，通过 yield 关键字和 next() 方法来交付和归还线程执行权，实现代码异步。 Generator 函数不同于普通函数，Generator 函数执行到yield关键字处会自动暂停，保护现场，返回一个遍历器（Iterator）对象，完成执行权的交付。之后，我们通过调用该遍历器对象的next()方法，回归现场，驱动 Generator 函数从断点处继续执行，完成执行权归还。归还执行权一般借助于 Promise 对象来实现。

        2. Async/Await Async/Await 语法本质上只是 Generator 函数的语法糖，支持更简洁更清晰的异步写法

        # 实例

        以请求一个网络数据为例

        1. 事件回调
        // 以jquery1.7为例
$.ajax({ 
	url: 'test.html',
	context: '请求数据',
	success: function(){
		// 请求成功的回调
	}
})
        1. 事件监听
        	var observe = new Observe()
	// 订阅事件
	observe.addListener('ajaxSucccess', function () {
		// 请求成功的回调
	})
	// 事件成功后发布
	observe.publish('ajaxSucccess', 'data')
        1. Promise模式
        	fetch('test.html')
		.then(function (res) {
			// 请求成功的回调
		})
        1. Generator模式
        	var getdata = function *() {
		yield fetch('test.html')
	}
	// 请求成功的数据
	getdata.next().value
        1. async模式
        	var getdata = async function () {
		var res = await fetch('test.html')
		// 请求成功后的操作，res为请求数据
	}
	getdata()
