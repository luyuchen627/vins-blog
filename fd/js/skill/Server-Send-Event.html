      # SSE

      server-send-event服务器推送，无需浏览器请求，就可推动到服务端，相比js的ajax轮询，节省请求；不同于websockt的基于全新协议的全双工通信，sse是基于http协议的单工通信

      # 使用

      # 客户端

      sse的连接由客户端的EventSource对象提供，url为请求地址，如果跨域设置withCredentials可以允许携带cookie 语法：new EventSource(url, option)

      sse连接之后，可以在EventSource实例上监听以下事件：(也可以使用addEventListener监听)

      • onopen 链接打开
      • onerror 链接出错
      • onclose 链接关闭
      • onmessage 接收数据

      sse也可以监听服务端的自定义事件 连接关闭： EventSource.close()

      # 服务端

      sse使用http协议，因此修改不大，修改一下响应头即可

      • Content-Type: text/event-stream
      • Cache-Control: no-cache
      • Connection: keep-alive

      sse对相应的数据结构有所规定，数据编码使用utf-8，或者二进制 每一条消息由一下字段组成：

      • event: 事件名称，默认是message, 修改即可自定义命令
      • data: 数据
      • id： 数据编号，作为断线重连的同步机制
      • retry: 重连时间

      注意：每个字段后边都必须有'/n',最后一个是有'/n/n'表示结尾

      var koa = require('koa');
var Router = require('koa-router');
var app = new koa();
var router = new Router();

router.get('/', function(ctx, next){
	ctx.set('Content-Type', 'text/event-stream');
	ctx.set('Cache-Control', 'no-cache');
	ctx.set('Connection', 'keep-alive');
	ctx.body = "data: The server time is" + new Date() + "\n\n";
});

app.use(router.routes());
app.listen(8080);
