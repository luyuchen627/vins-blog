                # 类型判断

                1. typeof typeof的返回值：
                • number
                • string
                • boolean
                • undefined
                • object (object, null, Math, array, set, map, 通过new命令生成的实例)都返回object
                • function (function, Class)返回function
                • symbol
                1. instanceof
                function instance_of(L, R) {  // L 表示左表达式，R 表示右表达式
	var O = R.prototype  // 取 R 的显示原型
	L = L.__proto__  // 取 L 的隐式原型
	while (true) { 
		if (L === null) 
			return false
		if (O === L) // 这里重点：当 O 严格等于 L 时，返回 true 
			return true
		L = L.__proto__  // 原型连寻找，因此可以判断子实例是否为父类型
	} 
}
                1. Object.prototype.toString 利用Object.prototype.toString.call(obj)的返回值做类型判断
                • String -> '[object String]'
                • Number -> '[object Number]'
                • Boolean -> '[object Boolean]'
                • Array -> '[object Array]'
                • Date -> '[object Date]'
                • Object -> '[object Object]'
                • RegExp -> '[object RegExp]'
                • Window -> '[object Window]'
                • HTMLDocument -> '[object HTMLDocument]'

                # 类型转换相关的函数

                valueOf和toString，这两个函数是定义在Object.prototype上的方法，会被继承，也可以改写

                1. valueOf是返回这个对象逻辑上对应的原始类型的值

                部分内置对象调用valueOf的行为：

                • String 字符串值 (原始类型)
                • Number 数字 (原始类型)
                • Boolean 布尔值 (原始类型)
                • Array 数组本身 (object)
                • Date 毫秒数 (原始类型)
                • Object 对象本身 (object)
                • Function 函数本身 (object)

                Math 和 Error 对象没有 valueOf 方法

                1. toString是返回这个对象的字符串描述方式

                部分内置对象调用toString的行为：

                • undefined 'undefined'
                • null 'null'
                • String 字符串值
                • Number 数字的字符串形式
                • Boolean
                  1. true -> 'true'
                  2. false -> 'false'
                • Array
                  1. 降为一维数组
                  2. 调用join(',')
                • Date 毫秒数
                • Object 对象本身的字符串表现 '[object Object]'
                • Function 函数本身的字符串表现 'function () { [native code] }'

                # 常见类型转换

                1. ToString() 参见类型转换相关的函数的toString

                2. ToBoolean() 当为undefined， null，Number(+0, -0, 或者NaN), 空字符串 返回false；其余情况返回true

                3. ToNumber()

                • undefined -> NaN
                • null -> 0
                • Boolean
                  • true -> 1
                  • false -> 0
                • String 类似于Number()
                • Symbol 抛出 TypeError 异常
                • Object ToPrimitive
                1. ToPrimitive() 转化为原始类型
                /** 将输入转化为原始类型
 * @params input 输入
 * @params PreferredType 期望的类型 可选 值：[ Number | String ]
*/

ToPrimitive(input [,PreferredType ])

                将input转化成一个原始类型的值。PreferredType参数要么不传入，要么是。如果PreferredType参数是Number，ToPrimitive这样执行：

                • 如果input本身就是原始类型，直接返回input

                • 如果PreferredType没有传值，input是内置的Date类型，PreferredType视为String，否则PreferredType视为Number，即对于所有非日期对象来说，对象到原始值的转换基本上是对象到数字的转换

                • PreferredType为Number,流程如下：

                1. 调用input.valueOf()，如果结果是原始类型，则返回这个结果
                2. 调用input.toString()，如果结果是原始类型，则返回这个结果
                3. 否则抛出TypeError异常
                • PreferredType为String,流程如下：
                1. 调用input.toString()，如果结果是原始类型，则返回这个结果
                2. 调用input.valueOf()，如果结果是原始类型，则返回这个结果
                3. 否则抛出TypeError异常

                # 隐式转换

                1. 运算符

                  1. 运算符+

                    1. 先把符号两边的值通过ToPrimitive转化为原始类型
                    2. 如果有一个值的类型为string，则将两个值通过ToString转化为string类型，做字符串拼接，否则通过ToNumber转化为number做算术运算

                    NaN 数学运算均返回 NaN

                  2. 其他运算符(-, *, /, %, 比较运算符)，把符号两边的值通过ToNumber转化为number做算术运算

                    比较运算符的 a>=b，有时会被处理成 !(a<b)

                    	var a = { k: 1 }
	var b = { k: 2 }
	a < b   // false, 两边调用ToPrimitive, 返回[object Object]两遍一致
	a == b  // false, 两边不同的引用
	a > b   // false, 同 a < b

	a <= b  // true, 处理成 !(a > b) 所以为true
	a >= b  // true, 同 a <= b

                2. x == y 比较 简单规则

                  1. x,y类型相同时，以下为特殊，其余均可以===比较 NaN == NaN 返回false
                  2. null == undefined -> true, 反之亦然
                  3. Number == String -> Number == ToNumber(String)，反之亦然
                  4. Boolean == y(any) -> ToNumber(Boolean) == y
                  5. x(any) == Boolean -> x == ToNumber(Boolean)
                  6. x(原始类型) == Object -> x == ToPrimitive(Object)，反之亦然

                如果==中有true或者false, 千万不要使用==(会被转为数字0,1来进行判断）

                如果两遍的值中有[]、''或者0，尽量不要使用 ==

                # 谜题

                1. {} + [] 和 [] + {}

                {} + []：运算符前面的{}认为是空的代码块，因此相当于 +[] = 0 [] + {}：即 ToNumber([]) + ToNumber({}) -> '' + '[object Object]'

                1. {} 和 !{} 与 [] 和 ![]

                {} == {} -> false {} == !{} -> {} == false -> {} == 0 -> '[object Object]' == 0

                [] == [] -> false [] == ![] -> [] == false -> [] == 0 -> '' == 0 -> true

                1. parseInt类型转化的问题
                /**
 * @params str 输入 字符串
 * @params base 转化的数字的进制 数字 默认：10
*/
parseInt(str, base)

                注意

                1. 当输入不是String类型时，会隐式转换，需要注意不可预料的问题
                2. 当输入小于**0.0000001(1e-7)**的数字转换成String时，会变成科学记号法
                parseInt(0.0000001) -> 1  // parseInt(0.0000001) == parseInt('1e-7')

parseInt(1/0, 19) -> 18   // parseInt(1/0, 19) == parseInt('Infinity', 19)

parseInt(false, 16) -> 250 // parseInt(false, 16) == parseInt('false', 16)

["1", "2", "3"].map(parseInt) -> [1, NaN, NaN]  
// 因为 parseInt 需要两个参数(val,radix)，其中 radix 表示解析时用的基数
// map 传了3个(element,index,array)，对应的 radix 不合法导致解析失败
