    # 数据库连接

    mongoose使用connect创建数据库连接

    	/*
	* url(s): 数据库地址,可以是多个,以`,`隔开
	* options: 可选,配置参数
	* callback: 可选,回调
	*/
	mongoose.connect(uri(s), [options], [callback])

    	// 连接单个数据库，返回默认的数据库连接Connection
	mongoose.connect('mongodb://用户名:密码@127.0.0.1:27017/数据库名称')
	// 连接多个数据库，使用createConnection创建多个数据库连接，返回当前数据库连接的实例
	const db = mongoose.createConnection('mongodb://用户名:密码@127.0.0.1:27017/数据库名称')

	// 连接监听
	db.once('error', () => console.log('Mongo connection error'))
	db.once('open', () => console.log('Mongo connection successed'))

    # Schema

    Schema一种以文件形式存储的数据库模型骨架，不具备对数据库的操作能力.类似于关系型数据库中的表结构，是表或者文档的抽象

    	new mongoose.Schema({字段...}, [options])

    # Schema的定义

    	const Schema = mongoose.Schema
	const ObjectId = Schema.Types.ObjectId

	const ArticleSchema = new Schema({
		title: {
			type: String,
			required: true
		},
		author: String,
		tag: {
			type: ObjectId,
			ref: 'Tag' // 关联TagCategory表的_id
		},
		createTime: {
			type: Date,
			default: Date.now
		}
	}, {
		connection: 'Article', // 模型名称,
	})

    字段定义的键值:

    • type： 类型
    • default: 默认值
    • ref: 关联文档
    • required: 不能为空
    • validate: 验证器
    • get: get方法
    • set: set方法
    • alias: 别名

    索引相关的键值

    • index: 索引
    • unique: 唯一索引
    • sparse: 稀疏索引

    字符串相关的验证

    • lowercase: 是否使用小写，为true会调用toLowerCase方法
    • uppercase: 是否使用小写，为true会调用toUpperCase方法
    • trim: 去除首尾空格，为true会调用trim方法
    • match: RegExp表达式
    • enum: 枚举，Array类型
    • minlength: 最小长度，数字类型
    • maxlength: 最大长度，数字类型

    数字相关的验证

    • min: 最小值
    • max: 最大值

    Date相关的验证

    • min: 最小日期，Date类型
    • max: 最大日期，Date类型

    每个Schema都会默认配置属性名为_id的字段作为主键，除非自己定义，方可覆盖

        var ObjectId = mongoose.Schema.Types.ObjectId
    var StudentSchema = new Schema({}) // 默认会有_id: ObjectId
    var TeacherSchema = new Schema({ id:ObjectId }) // 只有id: ObjectId

    所有的字段类型都定义在mongoose.Schema.Types

    基本类型

    • String 字符串
    • Number 数字
    • Date 日期
    • Buffer Buffer
    • Boolean 布尔类型
    • Mixed 混合类型，未指定的形势
    • ObjectId 索引
    • Array 数组

    # 虚拟字段

    使用Schema.virtual来定义虚拟字段，虚拟字段不写入数据库，每一个虚拟字段可以使用get方法来取值，set方法来设置相关的字段

    	const PersonSchema = new mongoose.Schema({
		name: {
			first: mongoose.Schema.Types.String,
			last: mongoose.Schema.Types.String
		}
	})
	// 定义名称的全称full, full字段不会写入数据库
    PersonSchema.virtual('name.full').get(function(){
		return this.name.first + ' ' + this.name.last
    })
	// 设置定义名称的全称时，修改first和last字段
    PersonSchema.virtual('name.full').set(function(name){
		var split = name.split(' ')
		this.name.first = split[0]
		this.name.last = split[1]
    })

    # Schema配置项options

    常用到的collection, 其他请参考官方文档

    • _id: true 唯一索引, 如果不需要,可以设置为false关闭
    • collection: 文档名称,在MongDB中默认使用Model的名字作为集合的名字，如过需要自定义集合的名字，可以通过设置这个选项
    • versionKey: '__v' 包含文档的内部修订,默认的是__v，修改相当于是给版本锁取名字
    • autoIndex 自动索引
    • capped: Number 上限设置,此属性对批量操作有效，用来限制一次对数据库操作的量
    • id: true mongoose分配给每一个schema一个虚拟属性id，它是一个getter。返回的是_id转换为字符串后的值
    • safe: true 安全属性（默认安全）
    • shardKey mongodb做分布式，才会使用该属性
    • strict: true 严格配置（默认启用）,保存时会严格检查字段类型，strict也可以设置为throw，表示出现问题将会抛出错误
    • toObject: 用来表示在提取数据的时候, 把documents 内容转化为Object内容输出. 一般而言只需要设置getters为true即可
    • toJSON： 该是和toObject一样的使用. 通常用来把 documents 转化为Object. 但是, 需要显示使用toJSON()方法,

    索引的设置也可以在定义模型的时候设置，例如Schema.index({ name: 1, type: -1 }); 1 表示正序, -1 表示逆序

    配置项也可以通过Schema.set方法来设置，例如：Schema.set('autoIndex', false); // 设置自动索引

    # Schema的扩展

    1. 静态方法 静态方法定义在Schema.statics的属性上，静态方法在Model层就能使用，也可以使用Schema.static('方法名', 具体方法)

    2. 实例方法 实例方法定义在Schema.methods的属性上，可以为后面的Model和Entity提供公共的属性，也可以使用Schema.method('方法名', 具体方法)

    # 中间件

    mongoose里的中间件,有两个, 一个是pre前置中间件, 一个是post后置中间件

    1. pre: 在指定方法执行之前绑定。 中间件的状态分为：parallel和series
    2. post: 相当于事件监听的绑定

    中间件一般仅仅只能限于在几个方法中使用 document 方法上:

    • init
    • validate
    • save
    • remove

    model方法上:

    • count
    • find
    • findOne
    • findOneAndRemove
    • findOneAndUpdate
    • update

    pre中间件有串行和并行之分，post没有，多个中间件时都需要通过next函数往下传递，而且需要注意顺序(与定义顺序有关)

    	// pre串行中间件
	schema.pre('save', function(next) {
		next() // 执行完毕，执行下一中间件
	})
	// pre并行中间件
	schema.pre('save', true, function(next, done) {
		next()
	})
	// post中间件
	schema.post('save', function(doc, next) {
		next()
	})

    # Model

    Model是由Schema编译而成的构造器，具有抽象属性和行为。Model的每一个实例（instance）就是一个document。document可以保存到数据库和对数据库进行操作

    语法:

    	mongoose.model('modelName', schema)

    # 查询

    查询语法：

    	/*  该语法适用于find, findOne, findById
	** conditions: 查询条件
	** fields: 提取的字段, 字符串形式：字段前有'-'表示不包含；对象形式：1为包含，0为不包含

	*/
	Model.findOne(conditions, [fields], [options], [callback])
    1. 直接查询 在查询时带有回调函数的，称之为直接查询，查询的条件往往通过API来设定
     PersonModel.findOne({'name.last':'dragon'},'some select',function(err,person){
      //如果err==null，则person就能取到数据
    });
    1. 链式查询
        Person
      .find({ name: vins })
      .where('age').gt(17).lt(66)
      .where('likes').in(['movie', 'draw'])
	  .skip(5)
      .limit(10)
      .sort('-age')
      .select('name age likes')
      .exec(callback)

    # model的子文档操作

    本来mongodb是没有关系的. 但是, mongoose提供了children字段. 让我们能够轻松的在表间建立关系. 现在,我们来创建一个子域:

    	var childSchema = new Schema({ name: 'string' });

	var parentSchema = new Schema({
	children: [childSchema]   //指明sub-doc的schema
	});
	// 在创建中指明doc
	var Parent = mongoose.model('Parent', parentSchema);
	var parent = new Parent({ children: [{ name: 'Matt' }, { name: 'Sarah' }] })
	parent.children[0].name = 'Matthew';
	parent.save(callback);

    现在, 我们就已经创建了3个table. 一个parent 包含了 两个child 另外,如果我们想要查询指定的doc。 则可以使用 id()方法. var doc = parent.children.id(id);

    子文档的CRUD, 实际上就是数组的操作, 比如push,unshift,remove,pop,shift等 parent.children.push({ name: 'Liesl' });

    mongoose还给移除提供了另外一个方法–remove: var doc = parent.children.id(id).remove();

    如果你忘记添加子文档的话，可以在外围添加, 但是字段必须在Schema中指定 var newdoc = parent.children.create({ name: 'Aaron' });

    # 更新

    	/*
	** conditions: 匹配条件
	** update: 更新的字段
	** options: 配置， 常用配置：{
		new: (boolean),   为true：返回update后的文档，false：返回update前的，默认是 false
		sort: null,       排序条件，与sort函数的参数一致。
		fields: null，    要返回的字段, 与find*的第二个参数一致。
		safe (boolean)    是否开启安全模式，默认是 true
		upsert (boolean)  如果没有匹配到内容,是否自动创建，默认是 false
		multi (boolean)   如果有多个doc，匹配到,是否一起更改 默认是 false
		strict (boolean)  使用严格模式，默认是 false
		overwrite (boolean) 匹配到指定doc，是否覆盖，默认是 false
		runValidators (boolean): 表示是否用来启用验证. 实际上,你首先需要写一个验证.默认是 false
	}
	** callback：回调
	*/
	Model.update(conditions, update, [options], [callback])
	// 查找并更新
	Model.findByIdAndUpdate(id, [update], [options], [callback])
	Model.findOneAndUpdate([conditions], [update], [options], [callback])
	// 实例更新
	document.set({})
	document.save()

    # 更新修改器:

    • $inc 增减修改器,只对数字有效. {'$set': {'age': 2 }} age自增2
    • $set 指定一个键的值,这个键不存在就创建它 {'$set': {'age': 2 }} age设置为2
    • $unset 删除一个建 {'$unset': {'age': 2 }} 删除age

    # 数组修改器:

    数组修改器用来修改字段类型为数组的字段

    • $push 给一个键push一个数组成员,键不存在会创建 Model.update({ age:22 }, { $push: { array: 10 }}) 执行后: 增加一个 array 键,类型为数组, 有一个成员10

    • $addToSet 向数组中添加一个元素,如果存在就不添加 执行前: array : [10] Model.update({ age:22 }, { $addToSet: { array: 10 }}) 执行后: array中有10所以不会添加

    • $each 遍历数组, 和 $push 修改器配合可以插入多个值 执行前: array : [10] Model.update({ age:22 }, { $push: { array: {$each: [1,2,3,4,5]} }}) 执行后: array : [10,1,2,3,4,5]

    • $pop 向数组中尾部删除一个元素 执行前: array : [10,1,2,3,4,5] Model.update({ age:22 }, { $pop: { array: 1 }}) 执行后: array : [10,1,2,3,4] tips: 将1改成-1可以删除数组首部元素

    • $pull 向数组中删除指定元素 执行前: array : [10,1,2,3,4] Model.update({ age:22 }, { $pull: { array: 10 }}) 执行后: array : [1,2,3,4] 匹配到array中的10后将其删除

    # 连表

    在定义Schema的时候，通过字段的ref指定相关联的Schema，来完成多个文档的关联， 该对象的字段为对象时为一对一的关联，查询文档时，没有找到关联会返回null 该对象的字段为数组时为一对多的关联，查询文档时，没有找到关联会返回[]

    	/*
	** path：类型：String或Object或Array。
		String类型时，指定要填充的关联字段，要填充多个关联字段可以以空格分隔。
		Object类型时，就是把 populate 的参数封装到一个对象里。
	** select: 类型：Object或String，可选，指定填充 document 中的哪些字段。
		Object类型时，格式如:{name: 1, _id: 0}, 为0表示不填充，为1时表示填充。
		String类型时，格式如:"name -_id"，用空格分隔字段，在字段名前加上-表示不填充
	** model: 类型：Model，可选，指定关联字段的 model，如果没有指定就会使用Schema的ref。多个数据库的关联必须指定关联的Schema
	** match: 类型：Object，可选，指定附加的查询条件。
	** options: 可选，指定附加的其他查询选项，如排序以及条数限制
	*/

	Query.populate(path, [select], [model], [match], [options])

    # 动态关联

    动态关联类似于js中的对象的引用

    	// Schema定义
	const commentSchema = new Schema({
	body: { type: String, required: true },
	on: {
		type: Schema.Types.ObjectId,
		required: true,
		refPath: 'onModel'
	},
	onModel: {
		type: String,
		required: true,
		enum: ['BlogPost', 'Product']
	}
	})

	// 创建model
	const Product = mongoose.model('Product', new Schema({ name: String }))
	const BlogPost = mongoose.model('BlogPost', new Schema({ title: String }))
	const Comment = mongoose.model('Comment', commentSchema)

	// 创建实例
	const book = await Product.create({ name: 'The Count of Monte Cristo' })
	const post = await BlogPost.create({ title: 'Top 10 French Novels' })

	const commentOnBook = await Comment.create({
	body: 'Great read',
	on: book._id,
	onModel: 'Product'
	})

	const commentOnPost = await Comment.create({
	body: 'Very informative',
	on: post._id,
	onModel: 'BlogPost'
	})

	const comments = await Comment.find().populate('on').sort({ body: 1 })
	comments[0].on.name // "The Count of Monte Cristo"
	comments[1].on.title // "Top 10 French Novels"

    # 虚拟填充

    使用虚拟填充可以添加数据库中不存在的字段，通过localField和foreignField可以自定义外键关联

    	const PersonSchema = new Schema({
	name: String,
	band: String
	})

	const BandSchema = new Schema({
	name: String
	})
BandSchema.virtual('members', {
  ref: 'Person', // 关联的Schema
  localField: 'name', // Find people where `localField`
  foreignField: 'band', // is equal to `foreignField`
  justOne: false,  // 如果justOne为真,将返回单个文档,否则是数组。justOne默认为false
  count: true // coun如果为true,则不返回文档，只返回文档数量
  options: { sort: { name: -1 }, limit: 5 }
})

const Person = mongoose.model('Person', PersonSchema)
const Band = mongoose.model('Band', BandSchema)

Band.find({}).populate('members').exec(function(error, bands) {
  /* `bands.members` is now an array of instances of `Person` */
})
