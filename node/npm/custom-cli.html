    # 命令解析

    命令解析库: commander

    常用操作

    1. version

    语法

    program.version('x.y.z')

    用于设置命令程序的版本号

    1. option

    语法

    program.option('-n, --name <name>', 'your name', 'GK')

    第一个参数是选项定义，分为短定义和长定义。用|，,，连接

    参数可以用<>或者[]修饰，前者意为必须参数，后者意为可选参数

    第二个参数为选项描述

    第三个参数为选项参数默认值，可选

    1. command

    语法

    program.command('init <path>', 'description')

    command的用法稍微复杂，原则上他可以接受三个参数，第一个为命令定义，第二个命令描述，第三个为命令辅助修饰对象

    第一个参数中可以使用<>或者[]修饰命令参数

    第二个参数可选

    当没有第二个参数时，commander.js将返回Command对象，若有第二个参数，将返回原型对象

    当带有第二个参数，并且没有显示调用action(fn)时，则将会使用子命令模式

    所谓子命令模式即，./pm，./pm-install，./pm-search等。这些子命令跟主命令在不同的文件中

    第三个参数一般不用，它可以设置是否显示的使用子命令模式

    1. description

    语法

    program.description('command description')

    用于设置命令的描述

    1. action

    语法

    program.action(fn)

    用于设置命令执行的相关回调。fn可以接受命令的参数为函数形参，顺序与 command 中定义的顺序一致

    1. parse

    语法

    program.parse(process.argv)

    此api一般是最后调用，用于解析process.argv

    示例

    program
   .version('0.0.1')
   .option('-i, --init <path>', 'init the project')
   .option('-t, --no-tests', 'test hook')

program
   .command('create <dir> [otherDirs...]')
   .description('create project commands')
   .action(function(dir, otherDirs) {
     console.log('dir "%s"', dir)
     if (otherDirs) {
       otherDirs.forEach(function (oDir) {
         console.log('dir "%s"', oDir)
       })
     }
   })

program.parse(process.argv)

    # github下载

    资源：download-git-repo

    语法

    download(url, dir, option, cb)

    url规则：

    • GitHub - github:owner/name or simply owner/name
    • GitLab - gitlab:owner/name
    • Bitbucket - bitbucket:owner/name

    dir：下载目录

    option:

    • clone 为true，则使用git clone下载，否则使用http下载，默认为false; 为true时，下载目录可以指定分支

    示例：

    // 下载vins-lu下面的test库的gh-pages分支到本地的test/tmp目录
download('github:vins-lu/test#gh-pages', 'test/tmp', function (err) {
	if (!err) {
		console.log('success')
	} else {
		console.log(err)
	}
})
