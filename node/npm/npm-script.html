              # npm 脚本

              npm 可以使用 npm run script 命令，运行在package.json文件里面使用scripts字段定义的脚本命令

              # 脚本传参

              向 npm 脚本传入参数，要使用 -- 标明

              脚本传参对命令组合，提供了方便

              "scripts": {
	"mode": "set ENV_MODE=test npm run",
	"test": "node test.js",
}

// shell运行
npm run mode -- test

              # 脚本参数

              在node文件中使用 process.argv 可以获取node脚本的参数，返回一个数组，第一个值表示node程序的执行路径，第二个表示执行文件的路径，后面的值表示获取到的命令的参数

              # 多个命令执行顺序

              • 顺序(继发)执行使用 &&
              • 并行执行使用 &
              "scripts": {
	"setmode": "set ENV_MODE=test",
	"test": "npm run setmode & node test.js", # 继发执行
}

// shell运行
npm run test

              # 前后挂钩

              • 前钩子：使用 pre- + script 定义命令后，在运行script前会运行此命令
              • 后钩子：使用 post- + script 定义命令后，在运行script后会运行此命令

              钩子命令对命令运行前后的变量设置或者清理工作提供了一些方便，可以使我们不用再单独写一个命令

              "scripts": {
	"pretest": "set ENV_MODE=test",
	"test": "node test.js",
}

// shell运行
npm run test

              # npm 变量

              1. 获取

              npm可以获取package.json对象里面的值，使用 npm_package_ + 对象的 key(嵌套使用 _ 链接)，获取相对应的值

              {
  "name": "npms",
  "version": "1.0.0",
  "description": "npms script",
  "main": "index.js",
  "scripts": {
	"setmode": "set ENV_MODE=test",
	"test": "npm run setmode & node test.js"
  },
  "config": {
	  "vins": "lu"
  },
  "author": "",
  "license": "ISC"
}

npm_package_name -> name -> 'npms'
npm_package_scripts_setmode -> 'set ENV_MODE=test'
npm_package_config_vins -> 'lu'

              在node文件中可以这样获取 process.env.npm_package_

              1. 设置

              使用 npm config set 可以设置package.json文件 config 里面的字段
