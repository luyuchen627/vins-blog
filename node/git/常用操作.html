# git常用操作

# 先介绍几个概念

  1. 工作目录

    工作目录就是我们在本地电脑上创建的文件夹，包含.git目录的文件夹

  2. 暂存区

    暂存区是储存了文件改动，待提交的地方，只有在暂存区存在的文件，本地仓库才会追踪到它的变化。暂存区位于.git文件夹中的index文件

  3. 本地仓库(版本库)

    本地仓库就是本地的代码版本库，包含所有提交版本的数据记录; .git中的HEAD指针，就指向仓库的最新版本

  4. 远程仓库

    远程仓库托管代码的服务器，例如github，gitlab

# 文件的状态

  1. 未修改 | 未跟踪

    未跟踪状态使用git add 跟踪文件改变

  2. 已修改，未暂存

    已修改状态表示修改了某个文件，但还没有提交保存

  3. 已暂存，未提交

    已暂存表示把已修改的文件放在了暂存区，等待下次提交

  4. 已提交，未推送

    已提交状态表示该文件已经被安全地保存在本地仓库

  5. 已推送

    文件推送到了远程仓库，团队可以拉取该修改

# 文件的检查（git diff）

  • git diff 检查工作区和暂存区之间的差异

  • git diff --cached 检查暂存区和本地仓库之间的差异

  • git diff master origin/master 检查本地仓库和远程仓库之间的差异

# 常用操作图解

# 提交

  1. git add . 文件添加
  2. git commit 文件提交
  3. git push [-u|f] origin master 推送

# 撤销修改

  1. 状态2（已修改，未暂存）
git checkout .  -> 回到状态1（未修改）
git checkout -- file  // 单个文件撤销
或者
git reset --hard  -> 回到状态1（未修改）

git reset --hard 操作等价于 git reset 和 git checkout 2步操作

  1. 状态3（已暂存，未提交）
git reset  -> 回到状态2（已修改，未暂存）
git reset HEAD file  // 单个文件撤销
git checkout .  -> 回到状态1（未修改）
或者
git reset --hard  -> 回到状态1（未修改）
  1. 状态4（已提交，未推送）
git reset HEAD^  -> 回到状态2（已修改，未暂存）
git reset --soft HEAD^  -> 回到状态3（已暂存，未提交）
git reset --hard origin/master  -> 回到状态1（未修改）
  1. 状态5（已推送）
git reset --hard HEAD^  -> 回到状态1（未修改）
git push -f  -> 推送到远程

# 版本回滚

  1. git reset --hard 版本id // 会删除提交记录 使用git reset回退公共远程分支的版本后，需要其他所有人手动用远程master分支覆盖本地master分支，不适合多人合作

  2. git revert 版本id

git revert HEAD      // 撤销最近一次提交
git revert HEAD~1    // 撤销上上次的提交，注意：数字从0开始
git revert 版本id    // 撤销0ffaacc这次提交

git revert 命令意思是撤销某次提交。它会产生一个新的提交，虽然代码回退了，但是版本依然是向前的，所以，当你用revert回退之后，所有人pull之后，他们的代码也自动的回退了

注意：

  • revert 是撤销一次提交，所以后面的版本id是你需要回滚到的版本的前一次提交
  • 使用revert HEAD是撤销最近的一次提交，如果你最近一次提交是用revert命令产生的，那么你再执行一次，就相当于撤销了上次的撤销操作，换句话说，你连续执行两次revert HEAD命令，就跟没执行是一样的
  • 使用revert HEAD~1 表示撤销最近2次提交，这个数字是从0开始的，如果你之前撤销过，那么也会计算在内的
  • 如果使用 revert 撤销的不是最近一次提交，那么一定会有代码冲突，需要你合并代码，合并代码只需要把当前的代码全部去掉，保留之前版本的代码就可以了
  • revert 命令的好处就是不会丢掉别人的提交，即使你撤销后覆盖了别人的提交，他更新代码后，可以在本地用 reset 向前回滚，找到自己的代码，然后拉一下分支，再回来合并上去就可以找回被你覆盖的提交了

# 分支操作

# 场景

  1. 新需求
  2. 环境分离
  3. 版本管理
  4. bug修复

鼓励频繁的使用分支来协同工作

# 分支的创建

git branch 分支name       // 创建
git checkout 分支name     // 切换，移动HEAD指针指向新创建的分支
or
git checkout -b 分支name [远程名]/[分支名]  // 创建本地相应的分支，同时从远程分支拉取分支
or > v1.6.2
git checkout --track origin/serverfix

远程分支 checkout 出来的本地分支，称为跟踪分支，跟踪分支可以使用git pull 和git push，master分支就是跟踪分支 git pull 是git fetch和git merge的合并命令

# 分支的删除

git branch -d 分支name   // 删除

# 分支的合并

git merge 分支name       // 合并

git rebase [主分支] [基分支] // 变基

合并冲突的时候，git会把文件标记为未合并状态，同时加入标准的冲突解决标记，手动解决冲突后使用add -> commit继续提交

# 分支管理

  1. git branch命令
  • 不带参数，列出当前所有分支
  • -v 查看各个分支最后一个提交对象的信息
  • --merged 查看哪些分支已被并入当前分支，列出的列表中没有 * 的分支通常都可以用 git branch -d 来删掉
  • --no-merged 查看尚未合并的分支
  1. 远程分支的拉取和推送
// 拉取
git fetch <远程仓库名> [远程分支名]:[本地分支名]
eg: git fetch origin a:local  // 拉取远程origin的a分支，并在本地创建local分支保存数据

// 推送
git push <远程仓库名> <分支名>
git push <远程仓库名> [本地分支名]:[远程分支名]  // 推送本地分支到远程，远程分支不存在会被创建
eg: git push origin local:a  // 把本地local分支推送到远程origin的a分支

// 删除
git push <远程仓库名> :<远程分支名>  // 删除远程分支，不要忘记":"

可以使用git remote add添加不止一个远程
