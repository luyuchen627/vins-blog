# gitignore规则

ignore文件用于忽略不想提交的文件

# 常用规则如下

  1. 以 # 开头的行会视为注释，被忽略

  2. 以 / 开头的表示目录，/ 模式匹配相对根目录下的该文件夹路径下的内容

  3. 以 / 结尾的模式，匹配相对于 .gitignore 文件所在的目录下的该文件夹路径下的内容

  4. 一个 * 匹配多个任意字符

  5. 两个 * 表示匹配任意中间目录

  6. ? 匹配一个任意字符

  7. [] 匹配任何一个列在方括号中的字符

  8. ! 取反，既不忽略

  9. 可以使用标准的glob模式匹配

注意：.gitignore只能忽略那些原来没有就没有不在缓存区的文件

# 实例

	#               表示此为注释，将被Git忽略
	*.a             表示忽略所有 .a 结尾的文件
	/node_modules   表示仅仅忽略项目根目录下的 node_modules 文件，不包括 a/node_modules
	build/          表示忽略 build/ 目录下的所有文件
	doc/*.txt       表示会忽略doc/notes.txt但不包括 doc/server/arch.txt

	**/foo:         表示忽略 /foo, a/foo, a/b/foo 等
	a/**/b:         表示忽略 a/b, a/x/b, a/x/y/b 等
	
	a/[abc].js      忽略 a/a.js , a/b.js, a/c.js

	config.php:     表示忽略当前路径的 config.php 文件
	
	a/*             忽略目录 a 下的全部内容；注意，不管是根目录下的 /a/ 目录，还是某个子目录 /child/a/ 目录，都会被忽略

	!a.config       不忽略 a.config 文件
